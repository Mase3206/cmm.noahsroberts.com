Dependencies

I do most of my development in macOS, which, for some unknown reason, has non-standard and out-of-date versions of basically every programming language (except Swift, of course). This can be fixed by installing more recent versions via Homebrew, but macOS will not switch the default global interpreter to the Homebrew version unless you do some frustrating tinkering. All this weirdness means that modules installed with pip can be installed in a billion different places, and it leads to a lot of "module is not installed" errors when trying to import things like Pandas (ugh) or PyYAML.

To fix this, I present: Dependencies! A simple Python program that can be interfaced with via command line or imported by another program to see if the specific interpreter and environment your program is running in has the modules it depends on to function (a.k.a. dependencies). If any are missing, it will automatically install them. I am not going to go into detail about how to use Dependencies via import or CLI here, because that's what the GitHub repository is for. You can find it at Mase3206/py-common-modules
